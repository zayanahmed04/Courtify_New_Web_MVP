# System Notifications

## Understanding Courtify Notifications

Courtify keeps you informed through automated notifications sent via email. These alerts notify you about booking updates, payment confirmations, and important platform information. This guide explains what notifications you'll receive and how to manage them.

## Types of Notifications

### Booking-Related Notifications

**New Booking Request (For Court Owners)**
- **Sent To**: Court owner
- **When**: Immediately when player creates a booking
- **Contains**: Player details, booking date/time, court name, total amount
- **Purpose**: Alert owner to approve or reject request
- **Action**: Owner logs in to respond to request

**Booking Approved (For Players)**
- **Sent To**: Player
- **When**: Court owner approves booking request
- **Contains**: Approval confirmation, booking details, payment instructions
- **Purpose**: Notify player to proceed with payment
- **Action**: Player completes advance payment

**Booking Confirmed (For Both)**
- **Sent To**: Player and court owner
- **When**: Payment successfully processed
- **Contains**: Confirmation, booking details, court address, next steps
- **Purpose**: Confirm booking is secured
- **Action**: Player prepares for their game

**Booking Rejected (For Players)**
- **Sent To**: Player
- **When**: Court owner declines booking request
- **Contains**: Rejection, reason (if provided), alternative suggestions
- **Purpose**: Notify player booking was declined
- **Action**: Player searches for alternative courts

**Booking Reminder (For Players)**
- **Sent To**: Player
- **When**: 24 hours before confirmed booking
- **Contains**: Booking details, court address, time countdown, checklist
- **Purpose**: Remind player of upcoming booking
- **Action**: Player prepares and confirms attendance

**Booking Cancelled (For Both)**
- **Sent To**: Both player and owner
- **When**: Booking is cancelled (player or owner initiated)
- **Contains**: Cancellation confirmation, refund info (if applicable), details
- **Purpose**: Confirm cancellation and refund status
- **Action**: Refund processing begins (if applicable)

### Payment Notifications

**Payment Successful**
- **Sent To**: Player
- **When**: Advance payment processed successfully
- **Contains**: Amount paid, transaction ID, receipt, booking confirmation
- **Purpose**: Confirm payment received and booking secured
- **Action**: None - payment complete

**Payment Failed**
- **Sent To**: Player
- **When**: Card declined or payment error occurs
- **Contains**: Failure reason, retry instructions, troubleshooting tips
- **Purpose**: Alert player payment didn't process
- **Action**: Player retries with correct card information

### Account Notifications

**Registration Confirmation**
- **Sent To**: New user
- **When**: Account created successfully
- **Contains**: Welcome message, account details, getting started guide
- **Purpose**: Confirm account created
- **Action**: User can log in and start booking

**Email Verification**
- **Sent To**: New user
- **When**: Account created (for email confirmation)
- **Contains**: Verification link, instructions
- **Purpose**: Verify email address is valid
- **Action**: User clicks link to verify email

**Password Reset**
- **Sent To**: User requesting password reset
- **When**: User clicks "Forgot Password"
- **Contains**: Password reset link, instructions
- **Purpose**: Allow user to reset forgotten password
- **Action**: User clicks link and creates new password

**Login Alert (Optional)**
- **Sent To**: User (if enabled in settings)
- **When**: Your account logs in from new device or location
- **Contains**: Login time, device info, location
- **Purpose**: Alert you to unauthorized access attempts
- **Action**: Monitor for suspicious logins

### Court Management Notifications (For Owners)

**Court Approved**
- **Sent To**: Court owner
- **When**: Admin approves new court listing
- **Contains**: Approval confirmation, court appears in listings, next steps
- **Purpose**: Confirm court is now visible to players
- **Action**: Owner can start receiving bookings

**Court Rejected**
- **Sent To**: Court owner
- **When**: Admin rejects court listing
- **Contains**: Rejection reasons, required fixes, resubmission instructions
- **Purpose**: Alert owner court needs corrections
- **Action**: Owner updates court and resubmits

**New Booking Review (Optional)**
- **Sent To**: Court owner
- **When**: Player leaves review/rating of court
- **Contains**: Rating, reviewer name, review text (if applicable)
- **Purpose**: Notify owner of customer feedback
- **Action**: Owner can respond to review

## Notification Channels

### Email Notifications

Primary notification method:
- Sent to email address on your account
- Check spam/junk folder if not seeing
- Contains all important updates
- Usually arrives within minutes
- Keep inbox uncluttered with filters/labels

**Email Features**:
- Professional formatting
- Clear subject lines
- Actionable information
- Booking details included
- Call-to-action buttons

### In-App Notifications

If using Courtify app or website:
- Push notifications on mobile app
- In-app alerts/banners
- Notification bell icon with count
- Dashboard notifications widget
- Real-time updates

## Managing Notification Preferences

### Accessing Notification Settings

**On Website/App**:
1. Log into your account
2. Go to "Settings" or "Preferences"
3. Find "Notifications" or "Email Preferences"
4. View all notification options
5. Enable/disable as desired

### Notification Options

Most notifications can be individually controlled:

**Booking Notifications**:
- ☐ New booking request received
- ☐ Booking approved
- ☐ Booking confirmed
- ☐ Booking rejected
- ☐ Booking cancelled
- ☐ 24-hour booking reminder

**Payment Notifications**:
- ☐ Payment successful
- ☐ Payment failed
- ☐ Payment receipt

**Account Notifications**:
- ☐ Login alerts
- ☐ Password changed
- ☐ Email changed
- ☐ Security alerts

**Marketing (Optional)**:
- ☐ New features announcement
- ☐ Promotional offers
- ☐ Court recommendations

### Customizing Preferences

Choose what you want to receive:
- **All Notifications**: Get all alerts
- **Essential Only**: Receive only critical booking/payment updates
- **None**: Disable most notifications (not recommended)
- **Custom**: Pick and choose individually

**Tips**:
- At minimum, keep booking and payment notifications ON
- Reminders are helpful for not missing bookings
- Consider disabling promotional if they annoy you
- Review settings regularly as needs change

## Notification Troubleshooting

### Not Receiving Notifications

**Check Email Settings**:
1. Verify email address in account is correct
2. Check spam/junk folder for Courtify emails
3. Add notifications@courtify.com to whitelist (if applicable)
4. Check email filters (may auto-delete)
5. Verify email account isn't full (may reject incoming)

**Check Platform Settings**:
1. Go to notification preferences
2. Verify notifications are enabled
3. Ensure email notifications checked ON
4. No filters set to disable
5. Save settings if made changes

**If Still Not Receiving**:
- Clear browser cache and refresh
- Log out and back in
- Try from different browser
- Check internet connection is stable
- Contact support if issue persists

### Too Many Notifications

If overwhelmed by notifications:

**Option 1 - Disable Some**:
- Go to notification settings
- Uncheck promotional/marketing options
- Keep only essential booking/payment alerts
- Customize to your preference

**Option 2 - Email Filters**:
- Create email filter/rule for Courtify
- Auto-organize into folder
- Set to not show as "unread"
- Review folder when convenient

**Option 3 - Digest Option**:
- If available, choose digest (daily summary)
- Receive one email with all updates
- Less frequent interruptions
- Still informed of important updates

### Duplicate Notifications

If receiving same notification multiple times:
1. Check email filters (may be duplicating)
2. Verify only one account registered
3. Check notification settings (may be double-enabled)
4. Contact support if issue continues

### Missed Notifications

If you missed a notification:

**Check Notification History**:
1. Log into account
2. Look for "Notification History" or "Activity Log"
3. View all recent notifications sent
4. Catch up on any missed items

**Take Action if Needed**:
- If booking approval received: Make payment
- If booking rejected: Search for alternatives
- If payment failed: Retry payment
- If reminder missed: Check if booking is today

## Important Notifications Not to Ignore

**Critical - Respond Immediately**:
- Booking approved (need to pay or booking lost)
- Payment failed (fix immediately to secure booking)
- Booking rejection (need alternative)

**Important - Respond Same Day**:
- Booking confirmed (verify details and prepare)
- Court owner message (respond if clarification needed)
- Login alert (verify if you recognize the device)

**Useful - Review This Week**:
- Booking reminder (verify plans still work)
- Review request (provide feedback)
- Promotional offers (take advantage if interested)

## Privacy and Notifications

### Your Information in Notifications
- Email notifications contain necessary booking info only
- No sensitive payment details in email
- Transaction ID provided but not full card number
- Personal information kept to minimum
- Courtify respects your privacy

### Unsubscribe Option
- Most emails include unsubscribe link
- Clicking unsubscribe disables all notifications
- NOT recommended - you'll miss important updates
- Instead, use settings to customize preferences
- Disable only specific notification types you don't want
